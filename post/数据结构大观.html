QUEUE队列

一种操作受限的线性表
特点：FIFO先进先出

常见操作

初始化 判空 判满 进队 出队

存储结构

顺序存储

正常队列

分配一块连续的存储单元存放队列中固定元素
front指向队首 rear指向队尾
出入队 两个指针都往前进
判空：Q.front == Q.rear(假溢出)

循环队列

用于解决假溢出的逻辑上首尾相接的队列
出入队 两个指针顺时针移动
出队：Q.front = (Q.front+1) % Maxsize
入队同理，区别在于取余的操作

判空判满的方法：

  • 牺牲一个存储单元：
    rear为进口，位置会先于front，当rear跑完一圈到front的后面的时候，使用一个存储单元，让其意识到下一个元素就是front,front和rear之间的元素已经被填满了
  • 增设size
  • 增设tag 删的时候为tag = 0，只有删才会队空。增的时候为tag = 1，只有增的时候会队满。

链式存储

是一个同时有队首指针和队尾指针的单链表
队空：Q.front = Q.rear
不存在队满和溢出的情况
front设置在队头 便于删除元素
rear设置在队尾 便于插入元素

双端队列

两端允许插入删除的线性表
输出受限：仅有一端允许输出
输入受限：仅有一端允许输入

出入队模拟 1，2，3，4的组合情况：

  • 输入受限：4，2，x，x 不可能出现

    因为输入受限，所以输出前的情况只能是1，2，3，4：2被夹在1，3中间，不可能先于1，3出

  • 输出受限：4，x，3，x不可能出现

    思考输出前的情况，1，2作为初始两个元素，一定是连在一起的，而因为输出受限，作为输出，也是连在一起，不可能被3断开

树和二叉树

树是一种递归的数据结构（一种逻辑结构同时也是分层结构）

相关概念

  • 树的二级结论
    • 结点数 = 总度数 + 1
    • 度为m的树：
      • 至少有一个结点的度为m，不能为空树
    • m叉树
      • 允许所有节点的度< m, 可以为空树
    • 已知高度为h
      • 的m叉树的最多结点(等比求和):$ \frac{m^h - 1}{m - 1} $
      • 的m叉树的最少结点：$h$
      • 的度为m的树的最少结点： $m + h - 1$
    • 具有n结点的m叉树的最小高度： $ \lceil \log_m(n(m-1)+1) \rceil $

      $\frac{m^h - 1}{m - 1} < n \le \frac{m^h-1}{m-1} $

  • 二叉树
    • 满二叉树 含有$2^h - 1$个结点
    • 完全二叉树 满二叉树基础上少任意个叶节点
  • 二叉树性质
    • $n_0 = n_2 + 1$
    • 完全二叉树最多只有一个度为1的结点 结点数为偶数：$n_1 = 1$ 奇数则$n_1 = 0$
  • 二叉树存储结构
    • 顺序存储 结点编号和完全二叉树对应
    • 链式存储 n个结点就有n+1个空链域

二叉树遍历 线索二叉树

  • 遍历方法
    • 前序 根左右
    • 中序 左根右
    • 后序 左右根
    • 层次（队列）
  • 二叉树的构造
    • 中序 + (前序 or 后序 or 层次)
  • 线索二叉树(方便遍历)
    • 线索：指向前驱/后序的指针
    • 存储结构：ltag rtag
    • 三种线索二叉树
      • 中序线索二叉树
      • 前序线索二叉树(不能找到前面的 除非能指向父节点)
      • 后序线索二叉树(不能找到后面的 即 不能遍历 除非能指向父节点)
    • 线索二叉树的确定：
      • 确定类型
      • 写出编号
      • 连上前驱后继
    • 二叉树线索化
      • 核心算法：对遍历的改造 + pre指针记录前驱结点
      • 注意最后一个结点的rchild和rtag的处理
      • 先序线索化有死循环的可能 注意线索化的条件要变为ltag == 0

树与森林

  • 存储结构
    • 双亲表示法 (顺序存储)
      • 用于寻找父结点 不适于找子结点
    • 孩子表示法 (顺序存储节点数据 结点保存链表头指针)
      • 用于找寻子结点 不适于找父结点
    • 孩子兄弟表示法 (二叉链表) 左孩子右兄弟
      • 化树为二叉树的重要方法
  • 遍历等价表
    树 森林 二叉树
    先根 先序 先序
    后根 中序(后根) 中序

哈夫曼树 并查集

哈夫曼树：

  • 概念：
    $结点的带权路径长度=根到结点的路径长度 \times 结点的权$
    $WPL = 树中所有叶子节点的带权路径长度之和$
    • 哈夫曼树（最优二叉树）：给定n个结点带权叶结点中 WPL最小的数
    • 哈夫曼树中 n个带权叶节点，有n-1个非叶节点 总结点个数为2n-1
  • 构造方法：循环寻找两个权值最小的数合并为一个新的结点（构造出来的树不唯一）
  • 编码方式：
    • 哈夫曼编码：频率作为权值构造哈夫曼树
    • 前缀编码：没有一个编码是另一个编码的前缀（是不是叫无前缀好一点？

      前缀编码至多:直接从最多位开始排列组合

    • 固定长度编码：每个字符用相等长度的二进制表示
    • 可变长度编码：

并查集：

  • 三要素：
    • 逻辑结构：一种简单的集合表示
    • 基本操作：
      • 初始化
      • find
      • union
    • 存储结构：顺序存储 双亲表示法
  • 优化：
    • Union：子树合并
    • find：路径压缩

有向无环图

用于描述表达式

AOV网

一个有向无环图的顶点组成的序列，满足每个顶点只出现一次且顶点A再序列中在顶点B的前面，就不存在顶点B直达顶点A的路径

拓扑排序的成功进行与否可以判断图中环的有无
通过栈实现拓扑排序。

性质：

  • 拓扑排序、逆排序可能不唯一
  • 图中有环就不存在拓扑排序

AOE网

一个顶点表示事件 边表示活动 边的权值表示完成该活动所消耗的时间
源点：入度为0的点，AOE网中仅有1个，表示工程的开始
汇点：出度为0的点，同样仅有一个，表示工程的结束

求解要点：所有事件的最迟最早发生时间，所有活动的最迟最早发生时间，所有活动的余量。

**关键活动(关键路径)**：余量为0的活动

查找

基本概念

查找：在数据集合中寻找满足某种条件的数据元素的过程
查找表： ⽤于查找的数据集合称为查找表，它由同⼀类型的数据元素（或记录）组成，有需要查找操作的静态查找表和可以查找/增删数据的动态查找表
关键字：数据元素中唯⼀标识该元素的某个数据项的值，使⽤基于关键字的查找，查找结果应该是唯⼀的

查找长度：在查找运算中，需要对⽐关键字的次数称为查找⻓度
平均查找长度ASL：所有查找过程中进⾏关键字的⽐较次数的平均值

三个基本查找

查找判定树 ：
成功查找的关键字对比次数 = 结点所在层数
失败的关键字对比次数 = 其父节点所在层数

顺序查找

算法实现：简单的顺序逐个查找，在逻辑尽头设计哨兵判断查找的完结，表中元素有序则可以提前终止查找，缩小ASL
时间复杂度：O(n)

折半查找

算法实现：有序的顺序表ONLY；low=0,high=len,mid = (low+high)/2,mid == key查找成功;mid != key,若在左,high = mid - 1,若在右,low = mid + 1;循环。

构造查找判定树，奇数个元素，左右部分相等，偶数个元素，向下取整，右边多1，向上则反。构造的结果一定是平衡二叉树。
折半查找的判定树，满足左<中<右，元素个数n时，树高$h = \lceil log_2(n+1) \rceil$

时间复杂度:O($log_2n$)

分块查找

算法实现：数据分快存储，块内无序，块间有序。有序的块组成索引表，记录最大关键字和区间，先折半/顺序查找索引表，再对分块内进行顺序查找
易错：对索引表进行折半查找时，若索引表中不包含目标关键字，则折半查找最终停在 low>high，要在 low所指分块中查找
ASL：索引表的ASL + 分块的ASL

设置n个元素，均分为b块，每块s个元素
顺序：$ASL = \frac {b+1}2 + \frac {s+1}2$
折半：$ASL = \lceil log_2(b+1) + \frac {s+1}2 \rceil$

树形查找

二叉排序树

BST：用于提高查找、插入和删除的速度
查找和插入比较简单，逐层向下就行
删除：叶节点直接删除；该节点有单个子树，子树代替即可；该节点有俩子树，则涉及到前驱和后继，使用前驱或后继结点代替。
查找效率：
时间复杂：best：$log_2n（平衡二叉树）$ worst: $n$
ASL:成功：$ASL = \frac {\sum C}{结点数}$ ；失败$ASL = \frac {\sum C}{空结点数}$

平衡二叉树

新元素：$平衡因子 = 左子树高 - 右子树高$
旋转操作：维护插入、删除导致的不平衡，均在最小不平衡树上进行
LL右单旋转root = 2：

将原根节点 root 的右子节点 newRoot 提升为新根。
原根节点的右子树的左子树（newRoot->left）成为原根节点的右子树。
更新旋转后两个节点的高度。

RR左单旋转root = -2：

将原根节点 root 的左子节点 newRoot 提升为新根。
原根节点的左子树的右子树（newRoot->right）成为原根节点的左子树。
更新旋转后两个节点的高度。

LR左右旋root0 = 2 root1 = -1:

对左子树进行 左旋（解决其右子树过高问题）。
对调整后的左子树和根节点进行 右旋（解决整体左子树过高问题）。

RL右左旋root0 = -2 root1 = 1:

对右子树进行 右旋（解决其左子树过高问题）。
对调整后的右子树和根节点进行 左旋（解决整体右子树过高问题）。

红黑树

性质小结：左根右 根叶黑 红不连 黑径同
每个结点有元素bh(路径上含有的黑节点的最多个数)
